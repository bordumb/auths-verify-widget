Auto-Resolve from Repository

Instead of supplying raw attestation JSON and a public key hex string, just point the widget at a repository URL. The widget automatically fetches identity and attestation data from the forge's Git refs.

GitHub Repository

Uses the repo attribute to auto-resolve from GitHub:

<auths-verify repo="https://github.com/bordumb/auths"></auths-verify>

Waiting...

With Tooltip Mode

Combine repo with display modes:

<auths-verify repo="https://github.com/bordumb/auths" mode="tooltip"></auths-verify>

Gitea Instance (Self-Hosted)

For Gitea or other self-hosted forges, auto-detection picks the right adapter:

<auths-verify repo="https://git.example.com/user/repo"></auths-verify>
(auto-verify off — no live server)

Forge Hint Override

Use forge to override auto-detection:

<auths-verify repo="https://git.custom.io/org/repo" forge="gitea"></auths-verify>

Identity Filter

Use identity to resolve a specific identity when a repo has multiple:

<auths-verify repo="https://github.com/bordumb/auths" identity="did:key:z6Mk..."></auths-verify>
